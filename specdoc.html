  • Membership

    • Paid user membership

      After arriving at the site the user ether decides to sign up for Folkrider on their own, or attempts to visit a portion of the site that is only available to members

      • User registration

        • Information gathering

          After deciding to become a Folkrider the user is directed to a form designed to gather their email, password, and desired alias. Upon 'failure of pass match', 'email address already registered', or 'unspecified failure condition', an alert message appears in the same window displaying the error thrown without having any destructive effect on entered information. Upon successful completion of registration form the users information will be entered into the user database.

        • Database management

          (mysql) Based primarily on email address with an id pass and alias bound to it

      • Transaction handling

        After successfully filling out the registration form the user is directed to a page where they can choose a method of payment, upon the successful completion of the transaction, their account is made active and they are directed to a thank you page.

        • Paypal
        • Amazon
        • Google wallet

    • Limited admin privileges for content providers

      After logging on the artist goes to a management page where they can edit the cd's and songs available in on their page

      • Change log / Audit trail

        A log that the administrator can easily check for errors or policy breaking modifications made by the content providers

  • In The Studio

    A place where work in progress can be featured by the artist and previewed by the members. Members olny area

  • Market

    A member visits the market section of the site, selects which products they want to purchase, add's them to a cart, and completes their order.

    • Catalog display

      • displaying in alphabetical order by artist

        Upon a member pressing the 'Sort by artist' button they are presented with a drop down list of every artist (broken ten to a page) in alphabetical order with a drop down each containing the artists folkrider catalog

      • displaying in alphabetical order by album

        Upon a member pressing the 'Sort by album' button they are presented with a list of every album (broken ten to a page) in the folkrider catalog sorted in alphabetical order

      • displaying by popularity

        Upon a member pressing the 'Sort by album' button they are presented with a list of every album (broken ten to a page) in the folkrider catalog sorted by order of popularity

      • catalog search function

        A member enters a string into a text field and presses the 'Find' button, the system preforms a search function on both artists album titles, if there is a result then it is returned, if nothing in the database matches then the user is directed to a page (still on the market page) stating that there are no products that match that search criteria

    • Shopping cart selection

      After finding the product the user wishes to perches they press an 'add to cart' button which triggers the database to store their selection associated with their user id and a time stamp, with functionality to ether transfer the order to the completed purchases table, or deprecate it after 24 hrs of inactivity. If there is an error, a nondestructive alert is thrown detailing the error.

    • Transaction handling

      After adding items to the shopping cart the user presses the checkout button, and finishes the payment method, at which point the cache is transferred from the active cart to the completed purchases.

      • Paypal
      • Amazon
      • Google wallet

  • Contact

    • General contact

      a non registered user fills out the contact form including name, email, subject, and body, if any field contains null or invalid value a non destructive function is called that displays an alert containing the error, if there are no errors than the message is processed and sent to a message log to be reviewed by the admin. the user is directed to a page thanking them for the message

    • Member contact

      a member fills out the contact form including name, (email being automatically pulled from the database), subject, and body, if any field contains null or invalid value a non destructive function is called that displays an alert containing the error, if there are no errors than the message is processed and sent to a message log to be reviewed by the admin. the user is directed to a page thanking them for the message

    • Artist Submission section

      After becoming a member an artist decides they want to have their work featured on the Folkrider network. they navigate to a page where they are presented with a form that asks for their name, the band name, email, reason they want to be a Folkrider Artist and a song they think is a fare representation of their work. If any of the data is invalid a non destructive function is called that displays an alert. If all fields are properly filled out then the message is sent to the content manager for review. The user is directed to a page thanking them for the message

  • Mp3 download functionality

    • As ZIP

      After the member presses the download link on a page a .zip file is sent to the users browser. If there are errors an alert is displayed informing the user of the issue along with possible fixes. if there are no errors the file finishes downloading with no further interaction with the user

    • As multi thread stream

      After the member presses the download link on a page the mp3's associated with that link are sent to the users browser. If there are errors an alert is displayed informing the user of the issue along with possible fixes. if there are no errors the file finishes downloading with no further interaction with the user

  • Folkrider radio

    • As a page

      A user navigates to the Folkrider radio page at which point Folkrider radio begins to stream.

    • As site wide widget

      upon navigating to Folkrider a prompt displays asking the user if they would like to listen to Folkrider radio for free while they browse the site, if the user answers no than the widget minimizes to an unobtrusive tab at the bottom of the page, if the user answers yes than a short message saying 'welcome to Folkrider, if you enjoy this station please consider becoming a Folkrider of only $12.00' and then begins to stream the radio station

  • Comment and rating system

    A member of the site after listening to an album, or multiple albums decides to leave a comment about an artist or album but filling out a form asking for the body of the comment and an optional rating, the alias and email are then retrieved from the session information, then submits all four pieces of information to a database that is then displayed on the artists or albums page
